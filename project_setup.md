Day 2 — Project Setup

Objective

The goal of this task was to create a clean and organized project structure for the dashboard UI project. A well-structured project helps in maintaining the code, improving readability, and making development easier as the project grows.

---

Project Folder Structure

The following folders were created to organize different parts of the project:

dashboard-ui-project/
│
├── pages/
│   └── dashboard.html
│
├── components/
│   └── sidebar.html
│
├── styles/
│   └── style.css
│
├── assets/
│   └── images/
│
└── index.html

Folder Description

/pages
This folder contains the main pages of the application such as the dashboard page and records page.

/components
This folder stores reusable UI components such as the sidebar, header, and navigation elements.

/styles
This folder contains all CSS files used for styling the project. The main stylesheet is "style.css".

/assets
This folder stores static resources like images, icons, and other media files used in the dashboard.

---

Base Files Created

index.html

This is the main entry page of the project. It serves as the starting point of the dashboard application.

style.css

This file contains the base styling for the project including layout design, fonts, colors, and spacing.

---

Running the Project Locally

To run the project locally:

1. Open the project folder on your computer.
2. Double-click the "index.html" file.
3. The dashboard project will open in the web browser.

---

Conclusion

Setting up a clean folder structure at the beginning helps keep the project organized. It makes it easier to manage pages, styles, components, and assets as the project becomes larger.
